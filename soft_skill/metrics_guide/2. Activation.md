Activation - 사용자가 프로덕트의 핵심 가치를 경험해서 활성화되는 단계이다.

경험하고 활성화 되는 것이 중요하다.

경험이 긍정 부정이야?, 얼마나 활성화 됬어? 라는 지표를 있어야한다.

## 1. 사용자 경험 지표

✅ 핵심 키워드 먼저 보기
체류시간 (DT: Dwell Time)

페이지 단위 체류시간

세션별 평균 체류시간

페이지뷰 (PV)

순방문자수 (UV)

UV당 평균 PV

페이지 이탈률 (반송률, Bounce Rate)

1. 체류시간 (Dwell Time, DT)
   사용자가 웹/앱에 머무른 시간을 측정

공식:
특정 기간 총 체류 시간 ÷ 방문자 수 = 평균 체류 시간

예시:

일주일간 체류시간 총합이 200분, 방문자가 2명 →
평균 체류시간 = 200분 ÷ 2 = 100분

2. 페이지 단위 체류시간
   특정 페이지 별로 평균 체류시간을 분석

공식:
해당 페이지 총 체류 시간 ÷ 해당 페이지 방문자 수

활용:

어떤 페이지가 사용자의 관심을 끌었는지 파악

페이지의 매력도와 유용성을 평가할 수 있다.

3. 세션별 평균 체류시간
   세션: 사용자의 "방문 → 활동 → 종료"까지를 의미

공식:
세션 종료시각 - 세션 시작시각 = 세션 시간

주의:

평균 세션 시간이 짧아진다면?

콘텐츠 재미 감소

페이지 로딩 지연

UX/UI 문제 가능성

이를 통해 사용자 경험(UX) 개선 포인트를 찾는다.

4. 페이지뷰 (PV: Page View)
   사용자가 웹/앱에서 열람한 페이지 총 개수

같은 페이지를 여러 번 방문하거나 새로고침해도 모두 카운트된다.

5. 순방문자수 (UV: Unique Visitor)
   일정 기간 동안 중복을 제외한 방문자 수

판별 방법: IP 주소, 쿠키, 로그인 정보 등 활용

6. UV당 평균 PV
   공식:
   페이지뷰 총합 ÷ UV 수

의미:

사용자 1명이 평균 몇 개의 페이지를 봤는지

사이트 몰입도, 흥미를 파악할 수 있음

7. 페이지 이탈률 (반송률, Bounce Rate)
   사용자가 첫 페이지만 보고 아무 추가 행동 없이 이탈한 비율

공식:
(반송 수 ÷ 전체 방문 수) × 100

주의:

반송률이 높을수록 좋지 않음

원인 분석 후 개선 필요

랜딩페이지 개선

첫 화면 콘텐츠 품질 향상

빠른 로딩 속도 제공

### 요약

| 지표                 | 설명                   | 주요 활용               |
| :------------------- | :--------------------- | :---------------------- |
| 체류시간 (DT)        | 전체 머무른 시간       | 유입 이후 관심도 파악   |
| 페이지 단위 체류시간 | 페이지별 머문 시간     | 페이지 매력도 분석      |
| 세션별 평균 체류시간 | 한 번 방문시 머문 시간 | UX/UI 문제 감지         |
| 페이지뷰 (PV)        | 본 페이지 총 개수      | 사이트 방문량 측정      |
| 순방문자수 (UV)      | 중복 없는 방문자 수    | 순수한 유입 파악        |
| UV당 평균 PV         | 1명당 본 평균 페이지수 | 사이트 몰입도 분석      |
| 반송률 (Bounce Rate) | 첫 페이지 이탈 비율    | 랜딩품질/초기 흥미 측정 |

## 활성화 지표

| 키워드              | 의미 요약                                                                             |
| ------------------- | ------------------------------------------------------------------------------------- |
| 활성화 지표         | 유입된 사용자가 핵심 가치(Aha Moment)나 전환(구매, 구독)까지 도달했는지 측정하는 지표 |
| 아하 모먼트         | 사용자가 제품의 핵심 가치를 처음으로 느끼는 순간                                      |
| 퍼널(Funnel) 전환율 | 사용자의 행동 단계를 구분하고, 각 단계에서 이탈/전환 비율을 확인하는 것               |
| CRM 채널            | 고객과 직접 소통하여 행동을 유도하는 수단(알림, 이메일 등)                            |
| UX 개선             | 사용자 경험을 향상시켜 이탈을 줄이는 방법                                             |
| 퍼널 단계 축소      | 단계를 간소화하여 최종 전환율을 높이는 전략                                           |

- 활성화란?
  - 유입된 사용자가 **서비스의 핵심가치(아하모먼트)를 경험**하거나
  - **구매/구독 같은 확실한 전환 행동**을 했을 때를 의미

---

- 활성화 기준 (서비스마다 다름)
  - **아하모먼트 경험 비율**
  - **구매수** 또는 **구매 전환율**
  - **구독수** 또는 **구독 전환율**
    > 아하모먼트란?
    >
    > 사용자가 "아 이 서비스 좋네!"라고 느끼는 결정적 경험을 의미한다.

---

- 퍼널(Funnel) 분석이 필요한 이유
  - 활성화 과정을 더 잘 이해하려면,
  - **사용자의 여정을 세부 단계**로 나누고 **각 단계 전환율**을 분석해야 한다.
  - 이를 **Funnel Conversion** 분석이라고 부른다.
  ### 🔹 Funnel 예시: 구매 퍼널
  ```
  웹사이트 방문 → 상품 리스트 조회 → 상품 조회 → 장바구니 담기 → 결제 시작 → 결제 완료
  ```
  | 단계                             | 전환율 |
  | -------------------------------- | ------ |
  | 웹사이트 방문 → 상품 리스트 조회 | 95%    |
  | 상품 리스트 조회 → 상품 조회     | 94.7%  |
  | 상품 조회 → 장바구니 담기        | 83.3%  |
  | 장바구니 담기 → 결제 시작        | 53.3%  |
  | 결제 시작 → 결제 완료            | 75%    |
  - 최종 전환율:
    `약 30% (100명 중 30명 결제 완료)`

---

- 최종 전환율을 높이는 방법
  ### 1. CRM 채널을 활용한 리마인드 & 유도
  - 특정 단계(예: 장바구니 담기 → 결제 시작)에서 이탈률이 높으면,
  - **CRM 채널** (이메일, 알림, 문자 등)로 유저에게 메시지를 보내 전환을 유도
  - **예시**
    - 장바구니에 담긴 상품이 **품절 임박** 알림
    - **한정 할인 쿠폰** 제공
    - **결제 마감 임박** 알림
  ### 2. UX/UI 개선
  - 결제 흐름을 간단하게 만들어 **사용자 불편을 최소화**하기
  - 페이지 로딩 속도 개선, 클릭 수 최소화 등
  - **예시**
    - 버튼을 눈에 띄게 개선
    - 결제 정보를 미리 저장하여 입력 단계를 줄임
  ### 3. 퍼널 단계 자체를 줄이기
  - **단계를 줄이면 이탈 포인트가 줄어든다.**
  - **예시**
    - **쿠팡**: 장바구니를 거치지 않고 상품 상세페이지에서 **바로 스와이프 결제** 가능
    - 결제까지 2단계로 축소 → 전환율 증가

### 요약

활성화 = 핵심가치 경험 or 확실한 행동(구매/구독)

Funnel Conversion 분석으로 활성화 흐름을 파악

최종 전환율을 높이는 3가지 방법:

CRM 메세지로 리마인드

UX/UI 개선

퍼널 단계 축소
